표준 입력 (stdin)

  • 파일 디스크립터: 0

표준 입력은 프로그램이 데이터를 입력받는 기본적인 방법이다 보통 사용자가 키보드를 통해 데이터를 입력하거나, 파일의 내용을 프로그램으로 전달할 때 사용한다.

[표준 입력 리다이렉션]

  • <(입력):명령어의 입력을 파일로부터 받습니다
grep "search_term" < file.txt

이 명령은 file.txt 파일의 내용을 읽어 grep 명령어로 검색합니다. "search_term"을 포함하는 줄을 출력합니다.

표준 출력 (stdout)

  • 파일 디스크립터: 1
  • 기본 대상: 터미널 화면

표준 출력은 프로그램이 처리 결과를 출력하는 기본적인 방법입니다.
보통 터미널에 결과를 표시하거나, 다음 프로그램에 데이터를 전달할 때 사용합니다

[표준 출력 리다이렉션]

  • >(덮어쓰기): 명령어의 출력을 지정한 파일에 저장합니다.
echo "hello world~" > out.txt

위의 명령어는 hello world를 out.txt 파일에 저장합니다 해당 파일이 이미 존재할 경우 파일을 덮어씁니다

  • >>(추가하기): 명령어의 출력을 지정한 파일에 추가합니다(기존 내용 유지)
echo "next level" >> out.txt

표준 에러 (stderr)

  • 파일 디스크립터: 2
  • 기본 대상: 터미널 화면

표준 에러는 프로그램 실행 중 발생하는 오류 메세지를 출력하는 방법이다.
표준 출력과는 별도로 처리 되며, 오류 메세지를 별도로 관리할 수 있도록 한다

[표준 에러 리다이렉션]

ls /undefined > result.txt 2>&1

2>&1는 표준 에러(파일 디스크립터2)를 표준 출력이 가리키는 위치로 리다이렉트 한다 (포인터 개념?)
해당 명령어를 실행하면 ls명령어의 결과와 표준 에러 모두 result.txt에 저장된다

리다이렉션은 표준 입력,출력,에러를 다른 명령어는 파일로 전달하는 것을 말함

파이프

파이프는 한 명령어의 출력을 다른 명령어의 입력으로 연결한다 | 기호를 사용하여 연결.

ps -ef | grep "java"
